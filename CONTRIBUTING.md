# Contributing to Cinematic Blueprint

Thank you for your interest in contributing to Cinematic Blueprint! This is an open source project and we welcome contributions from the community.

## Ways to Contribute

- **Bug Reports** — Found a bug? Open an issue with steps to reproduce
- **Feature Requests** — Have an idea? Open an issue to discuss
- **Code Contributions** — Submit a pull request
- **Documentation** — Improve docs, fix typos, add examples
- **Templates** — Create new story structure templates

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Open `cinematic-blueprint.html` in your browser — no build step required!
4. Make your changes
5. Test in multiple browsers (Chrome, Firefox, Safari)
6. Submit a pull request

## Project Structure

```
├── cinematic-blueprint.html   # Main app (single-file HTML/CSS/JS)
├── templates/                 # Story structure templates
├── docs/                      # Technical documentation
└── story-cards-mcp/           # MCP server for AI integration
```

## Code Style

- This is a single-file HTML application — keep it that way
- Use vanilla JavaScript (no frameworks)
- Follow existing code formatting
- Test accessibility (keyboard navigation, screen readers)

## Pull Request Guidelines

1. **One feature per PR** — Keep changes focused
2. **Test thoroughly** — Check all three views (Acts, Swimlanes, Storyboard)
3. **Update docs** — If your change affects usage, update the README
4. **Describe your changes** — Clear PR description helps reviewers

## Reporting Bugs

When reporting bugs, please include:
- Browser and version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
